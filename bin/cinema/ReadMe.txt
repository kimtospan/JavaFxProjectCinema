Εισαγωγή

Στο πλαίσιο αυτής της εργασίας θα πρέπει να αναπτυχθεί μια εφαρμογή για τη διαχείριση
ενός πολυχώρου. Ο χώρος αυτός παρέχει τη δυνατότητα στους πελάτες του να
παρακολουθήσουν κάποια από τις προβολές ταινιών του ή να διοργανώσουν παιδικά πάρτι
όπου τα παιδιά θα έχουν τη δυνατότητα να παρακολουθήσουν μια ταινία με συνοδεία
ειδικών μενού αλλά και άλλων δραστηριοτήτων (πχ. κόψιμο τούρτας/συμμετοχή κλόουν ή
ανιματέρ).

H εφαρμογή διαχείρισης του πολυχώρου
Η εφαρμογή που καλείστε να αναπτύξετε θα επιτρέπει σε υπαλλήλους του πολυχώρου μέσα
από κατάλληλο γραφικό περιβάλλον να καταχωρούν και να διαχειρίζονται τα αιτήματα των
πελατών, τα περισσότερα εκ των οποίων θα πραγματοποιούνται τηλεφωνικά. Ο
χρήστης/υπάλληλος μέσα από το κεντρικό παράθυρο της εφαρμογής θα μπορεί να επιλέξει
αν θέλει να διαχειριστεί τις διάφορες εκδηλώσεις/events (απλές προβολές ταινιών ή παιδικά
πάρτι), να κάνει κρατήσεις (κράτηση χώρου για παιδικό πάρτι/έκδοση εισιτηρίων για
προβολές) και να αναζητεί εκδηλώσεις με διαφορετικά κριτήρια.
Για κάθε εκδήλωση (προβολή ταινίας ή παιδικό πάρτι) που θα γίνεται στον πολυχώρο θα
πρέπει να καταγράφεται η ταινία που θα προβληθεί, η ημερομηνία και η ώρα προβολής
καθώς και η αίθουσα που θα γίνει η προβολή.

Για κάθε ταινία που προβάλλεται, η εφαρμογή θα πρέπει να έχει καταχωρημένες
λεπτομέρειες της, όπως ένας μοναδικός κωδικός που την χαρακτηρίζει, ο τίτλος της, το έτος
παραγωγής της, οι βασικοί ηθοποιοί που συμμετέχουν καθώς και η διάρκεια της σε λεπτά.
Αντίστοιχα και για κάθε αίθουσα που είναι διαθέσιμη για προβολή θα πρέπει η εφαρμογή να
έχει αποθηκευμένα στοιχεία όπως ένας κωδικός και μια περιγραφή που την χαρακτηρίζει,
καθώς και τον αριθμό των σειρών και των στηλών των θέσεων που διαθέτει. Για ευκολία
στην υλοποίηση θεωρήστε ότι όλες οι αίθουσες έχουν τον ίδιο αριθμό καθισμάτων με την
ίδια ακριβώς διάταξη.

Ειδικά για κάθε εκδήλωση που διοργανώνεται από τον πολυχώρο και που αφορά
αποκλειστικά την προβολή ταινιών, για να είναι δυνατή η κράτηση θέσεων και η έκδοση
εισιτηρίων θα πρέπει να διατηρείται ο αριθμός των συνολικών θέσεων της αίθουσας που θα
γίνει η προβολή, ο αριθμός των κατειλημμένων θέσεων καθώς και μια δομή δεδομένων στην
οποία θα αποθηκεύονται οι συγκεκριμένες θέσεις που είναι ήδη κατειλημμένες (σειρά και
στήλη/θέση ανά κράτηση/εισιτήριο). Αντιθέτως για κάθε εκδήλωση που αφορά την
διοργάνωση παιδικού πάρτι θα πρέπει να καταγράφεται η ημερομηνία και η ώρα δέσμευσης
της αίθουσας, ο αριθμός των παιδιών που θα συμμετάσχουν, αν απαιτείται ειδικό παιδικό
μενού ή κάτι πιο απλό όπως ποπ κόρν και αναψυκτικό, αν θα συμπεριλαμβάνονται υπηρεσίες
όπως κόψιμο τούρτας, συμμετοχή κλόουν/ανιματέρ, κλπ.

Και για τις 2 εκδηλώσεις υπάρχει σχετική πληρωμή. Με άλλο τρόπο υπολογίζεται η
πληρωμή ενός εισιτηρίου (διαφορετική τιμή ανά είδος εισιτηρίου) για προβολή ταινίας με
άλλο τρόπο καθορίζεται η τιμή σε μια εκδήλωση διοργάνωσης πάρτι (τιμή για δέσμευση
αίθουσας, για ειδικό μενού ή για απλό μενού, για κοπή τούρτας ή για ύπαρξη
κλόουν/ανιματέρ). Ο υπολογισμός της τιμής της εκδήλωσης θα γίνεται αυτόματα (καθορίστε
εσείς τον τιμοκατάλογο) και θα εμφανίζεται σε σχετικό πεδίο στην οθόνη, το οποίο δεν θα
πρέπει να έχει τη δυνατότητα ο χρήστης να το μεταβάλλει.
Μέσα από κατάλληλο γραφικό περιβάλλον ο χρήστης θα πρέπει να έχει τη δυνατότητα να
εκτελεί τις ακόλουθες λειτουργίες:

Διαχείριση εισιτηρίων για προβολή ταινιών
▪ Έκδοση νέου εισιτηρίου: Ο χρήστης θα μπορεί να εκδίδει ένα νέο εισιτήριο. Σε κάθε
εισιτήριο θα πρέπει να αναγράφεται ένας μοναδικός κωδικός, η προβολή για την οποία
εκδίδεται, η ημερομηνία έκδοσης του, το είδος του εισιτηρίου, αν δηλαδή είναι (κανονικό,
φοιτητικό ή πολυτέκνων, κλπ), η θέση που θα γίνει η κράτηση και η τιμή του.
▪ Εμφάνιση λίστας εισιτηρίων: Θα εμφανίζεται η λίστα με τα εισιτήρια που έχουν εκδοθεί για
συγκεκριμένη προβολή.
▪ Τροποποίηση στοιχείων εισιτηρίου: Με αυτή την επιλογή ο υπάλληλος θα μπορεί να αλλάξει
όλα τα στοιχεία ενός εισιτηρίου εκτός από τον κωδικό του. (Bonus: Επιτρέψτε να γίνεται
τροποποίηση του εισιτηρίου μέχρι και 2 ώρες πριν από την προβολή).
▪ Ακύρωση εισιτηρίου: Ο χρήστης θα μπορεί να επιλέγει ένα εισιτήριο και να το ακυρώνει
χωρίς χρέωση. (Bonus: Επιτρέψτε να γίνεται ακύρωση του εισιτηρίου μέχρι και 2 ώρες πριν
την έναρξη της προβολής).

Διαχείριση των παιδικών πάρτι
Αντίστοιχες θα είναι και οι λειτουργίες για κάθε παιδικό πάρτι που θα διοργανώνει ο
πολυχώρος.
▪ Καταχώριση νέου πάρτι: Ο χρήστης θα μπορεί να κάνει κράτηση ενός χώρου για ένα νέο
πάρτι. Κάθε νέο πάρτι που διοργανώνεται χαρακτηρίζεται από έναν μοναδικό κωδικό. Ο
χρήστης θα πρέπει να καταχωρεί όλα τα στοιχεία που απαιτούνται για τη διοργάνωση του
πάρτι, όπως αναφέρθηκε προηγουμένως.
▪ Εμφάνιση λίστας πάρτι: Θα εμφανίζεται η λίστα με τα πάρτι που έχουν
χρονοπρογραμματιστεί για συγκεκριμένο χρονικό διάστημα.
▪ Τροποποίηση στοιχείων παιδικού πάρτι: Με αυτή την επιλογή ο υπάλληλος θα μπορεί να
αλλάξει όλα τα στοιχεία ενός πάρτι εκτός από τον κωδικό του. (Bonus: Τροποποίηση θα
μπορεί να γίνει μέχρι και 1 μέρα πριν την προγραμματισμένη ημερομηνία).
▪ Ακύρωση πάρτι: Ο χρήστης θα μπορεί να επιλέγει ένα παιδικό πάρτι για το οποίο έχει γίνει
κράτηση και να το ακυρώνει χωρίς χρέωση. (Bonus: Επιτρέψτε να γίνεται ακύρωση του
πάρτι μέχρι και 1 μέρα πριν την προγραμματισμένη ημερομηνία).
Αναφορές (Reports) σχετικά με τις εκδηλώσεις
Με την επιλογή αυτή ο υπάλληλος θα μπορεί να αναζητά:
▪ Για συγκεκριμένη ημέρα τις προβολές ταινιών και τα παιδικά πάρτι που έχουν
προγραμματιστεί, εμφανίζοντας τα βασικά στοιχεία τους.
▪ Τα έσοδα που θα αποκομίσει ο πολυχώρος από την διοργάνωση παιδικών πάρτι για
συγκεκριμένο χρονικό διάστημα που θα ορίσει ο υπάλληλος.
▪ Τα έσοδα που θα αποκομίσει ο πολυχώρος από την κράτηση εισιτηρίων για προβολές
ταινιών συγκεκριμένης ημέρας.

Γενικές Υποδείξεις
▪ Η εργασία μπορεί να εκπονηθεί από ομάδα φοιτητών (το πολύ μέχρι 2 φοιτητές).
▪ Κατά την εκκίνηση του προγράμματος θα πραγματοποιείται αρχικοποίηση της εφαρμογής.
Πιο συγκεκριμένα, θα πρέπει να δημιουργούνται συγκεκριμένες ταινίες, αίθουσες του
πολυχώρου και κάποιες προβολές ταινιών τα οποία στη συνέχεια θα μπορεί να
χρησιμοποιήσει ο χρήστης/υπάλληλος της εφαρμογής για να καταχωρίσει εισιτήρια και να
προγραμματίσει παιδικά πάρτι. Οι πληροφορίες για τις ταινίες, τις αίθουσες και τις προβολές
ταινιών που θα αρχικοποιούν την εφαρμογή θα πρέπει να υπάρχουν καταχωρημένες σε
αντίστοιχα αρχεία κειμένου, τα οποία θα τα διαβάζει η εφαρμογή κατά την εκκίνηση της.
▪ Μπορείτε να κάνετε τις δικές σας σχεδιαστικές επιλογές αν κάτι δεν αναφέρεται επακριβώς
στην εργασία.
▪ Σε όλες τις περιπτώσεις εισαγωγής δεδομένων από το χρήστη θα πρέπει να
πραγματοποιούνται έλεγχοι εγκυρότητας των δεδομένων (πχ. αν δεν έχουν καταχωρηθεί
στοιχεία όπου απαιτείται). Σκόπιμο είναι οι έλεγχοι αυτοί να γίνουν με τη χρήση εξαιρέσεων
(Custom Exceptions).
▪ Για τη σχεδίαση του γραφικού περιβάλλοντος δεν θα πρέπει να γίνει χρήση έτοιμων
εργαλείων (π.χ. δυνατότητα σχεδίασης του Netbeans). Θα πρέπει να υλοποιήσετε σχετικό
κώδικα είτε με την χρήση JavaFX (θα γίνει παρουσίαση των βασικών στοιχείων της
βιβλιοθήκης στο εργαστήριο) είτε με την χρήση Swing (θα ανέβει σχετικό υλικό στο eclass).
